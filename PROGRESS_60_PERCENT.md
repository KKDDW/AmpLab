## 💡 核心特性

### 1. 完整的数据流
```
用户操作 → UI 组件 → 回调 → dispatcher → config/数据
   ↓                                          ↓
事件通知 ← 事件总线 ← 后端计算 ← ← ← ← ← ← ← ↓
   ↓
UI 更新（结果表格、日志）
```

### 2. 组件化设计
- 每个面板都是独立的类
- 可以单独测试
- 易于维护和扩展

### 3. 双向数据绑定
- UI 修改 → 自动保存到 config
- Config 变化 → 自动更新 UI
- 防止递归更新

### 4. 实时更新
- 计算结果实时显示
- 日志实时追加
- 状态颜色实时标记
